# Stub elimination via parent API delegation

Loop 7 strategy for reducing `search not yet implemented` scaffolds without duplicating reverse-engineering work.

## Problem

The monorepo has **194** CLIs (`*-cli/`) covering **321** brands (`scripts/groups.json`). Most slugs still ship a scaffold `internal/client/search.go` until a worker implements the real endpoint.

Some slugs are **separate agent entrypoints** (their own binary name) but share a **parent booking API** already implemented on another slug — e.g. Design Hotels → Marriott Bonvoy search, Leonardo → Radisson.

## Coordination artifact: `scripts/groups.json`

Push updates to this file **before** parallel parent-API agents merge, so everyone agrees on slug → parent mapping.

```json
{
  "version": 1,
  "parent_overrides": {
    "designhotels": "marriott",
    "leonardo": "radisson",
    "iberojet": "tui"
  },
  "groups": [
    {
      "slug": "designhotels",
      "parent": "marriott",
      "name": "Design Hotels",
      "cat": "hotel",
      "brands": ["Design Hotels"]
    }
  ],
  "brand_parents": {
    "Design Hotels": "marriott",
    "Gran Meliá": "melia"
  }
}
```

| Field | Meaning |
|-------|---------|
| `groups[].slug` | CLI directory / binary name |
| `groups[].parent` | Slug that owns the shared search API |
| `parent_overrides` | Explicit slug→parent when `parent ≠ slug` |
| `brand_parents` | Every brand name → parent slug (auto-generated) |

When `parent === slug`, the CLI must implement (or receive) its own API — wiring is a no-op until the parent search exists on that same slug.

## Tooling

| Script | Role |
|--------|------|
| `scripts/groups_lib.py` | Load/save groups.json, parent overrides, `brand_parents` |
| `scripts/scaffold-clis.py` | Reads groups.json; writes `parent` on regen |
| `scripts/wire-stub-to-parent.py` | Replace stub `search.go` with parent delegate |

### Wire algorithm

For each `{slug}-cli`:

1. `internal/client/search.go` contains `search not yet implemented` (stub), or `--force` to refresh an existing wired file.
2. `groups.json` sets `parent = P` with **P ≠ slug**.
3. `{P}-cli/internal/client/search.go` is **implemented** (no stub marker).
4. Replace stub using one of:
   - **`exec` (default):** generated `search.go` runs `../{P}-cli/{P} search --json --brand …` and decodes JSON. Avoids Go `internal/` import restrictions across modules.
   - **`copy-client`:** copy parent `internal/client/{search,read,…}.go` into child and set child `BaseURL` to parent's URL.
5. No `go.mod` dependency on the parent module is required for `exec`.

Already-wired files contain `Code generated by scripts/wire-stub-to-parent.py` and are skipped unless `--force`.

### Commands

```bash
# Refresh brand_parents + parent fields (safe to run anytime)
python3 scripts/wire-stub-to-parent.py --sync-groups

# Dry-run: list wireable stubs
python3 scripts/wire-stub-to-parent.py --dry-run

# Wire all eligible stubs (after parent APIs land)
python3 scripts/wire-stub-to-parent.py --verify

# Re-generate wired delegates (e.g. after switching exec → copy-client)
python3 scripts/wire-stub-to-parent.py --force --verify
```

`--verify` runs `go mod tidy` + `go build` on each newly wired slug.

## Delegation vs symlink

| Strategy | When to use |
|----------|-------------|
| `exec` | Default. Child binary delegates at runtime to parent binary with `--brand`. Builds without cross-module `internal/` imports. |
| `copy-client` | Copy parent `client/*.go` into child package; patch child `BaseURL`. Use when exec is undesirable (no parent binary on PATH). |

Symlinks are not committed (portability); `copy-client` is the file-level equivalent.

## When *not* to wire

- `parent === slug` — wait for direct API implementation on that slug.
- Parent search is still a stub — wiring would delegate to another stub.
- Child already has a bespoke implementation (e.g. `mamashelter`, `25hours` use their own homepage parsers despite Accor ownership).
- Airlines with unrelated APIs (do not wire `canaryfly` → `binter`; different operators).

## Adding a new parent mapping

1. Add `"child_slug": "parent_slug"` to `parent_overrides` in `scripts/groups_lib.py` (or `groups.json`).
2. Commit & push `groups.json` so parallel agents know the target parent.
3. After parent API merges, run `python3 scripts/wire-stub-to-parent.py --verify`.
4. Update counts in this doc.

## Wire a stub to its parent

```bash
python3 scripts/wire-stub-to-parent.py --slug designhotels --parent marriott
python3 scripts/wire-stub-to-parent.py --slug designhotels --parent marriott --write
```

Never run `python3 scripts/scaffold-clis.py` on a dirty tree — it overwrites live search.go files.
